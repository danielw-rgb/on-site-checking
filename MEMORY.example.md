# MEMORY — Onsite Checking Workflow

A running log of items, quirks, and **special requirements** discovered while using this repo.
Use this to remember things that aren't obvious from the code or `CLAUDE.md` — client-specific
rules, recurring gotchas, per-site exceptions, and decisions to apply on future runs.

> **This is a template.** `MEMORY.md` itself is **personal and gitignored** — it holds
> client-confidential notes (agency accounts, properties, findings) and is never committed, so a
> fresh clone won't carry anyone else's. On first use, copy this file to `MEMORY.md` and start
> logging. Each person keeps their own.

> **How to use:** When you (or Claude) hit something worth remembering during a check, add a
> dated bullet under the right section. Keep each entry to one fact. Convert "last week" /
> "next run" into actual dates. Prune entries that turn out to be wrong.
>
> **Entry format:** `- [YYYY-MM-DD] <site or scope> — <the fact / requirement>. Why: <reason>.`
